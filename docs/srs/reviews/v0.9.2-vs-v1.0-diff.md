### 관점 1 요약

- **단순화 벡터**: 3-provider → 1-provider (Gemini). MVP 구현 부담·테스트·번들 감소.
- **명확화 벡터**: REQ-NF-011 daily-digest 구현 방식 명시(Loops List Send 위임) · REQ-NF-122 2단 임계.
- **경로 개방성 유지**: facade 구조 보존 → Phase 5.5+ OQ-R17 결과에 따라 provider 확장 무중단 가능.
