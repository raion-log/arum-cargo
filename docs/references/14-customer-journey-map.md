# Customer Journey Map (CJM) — 고객 여정지도

> **목적**: 페르소나가 RAION을 처음 접한 순간부터 재방문·재구독까지 겪는 **행동·생각·감정·Pain·개선기회**를 단계별로 시각화한다.
> **원칙**: "예쁘면 다 좋은 게 아니라, 지식으로서 가치있는 부분을 가독성 높고 재사용하기 쉽게 추출하기." — 이 표는 PRD 기능 우선순위의 **재료**다.
> **방법론 출처**: 고객 여정지도(CJM) 표준 구조 = `단계 / 고객 행동 / 고객 생각 / 감정 / Pain Point / 개선 기회`

---

## 1. CJM 표준 단계 (공통)

| 단계 | 설명 |
|---|---|
| 1. 인지 (Awareness) | RAION의 존재를 처음 알게 되는 순간 |
| 2. 탐색 (Exploration) | 랜딩 페이지·소개글을 훑어보며 가치 판단 |
| 3. 첫 사용 (First Use) | 뉴스·채용·운항 카드를 실제로 클릭·스크롤 |
| 4. 전환 (Conversion) | 이메일 구독·인증 완료 |
| 5. 유지 (Retention) | 일일 다이제스트 수신·재방문 루프 |
| 6. 확산 / 이탈 (Advocacy / Churn) | 추천·공유하거나 또는 구독 해지 |

---

## 2. Core vs Extreme 병렬 비교 — C1 김지원 vs E2 이준서

> **이 비교는 UX 설계의 "양극단 스트레스 테스트"다.**
> 김지원(승무원 취준생, 기술 중상)과 이준서(17세 항공 덕후, 기술 최상)의 여정을 나란히 놓으면, **둘 다 만족시키는 화면 = 나머지 10명도 만족시키는 화면**이 된다.

### 2.1 C1. 김지원 (Core) — 승무원 취준생, 23세

| 단계 | 행동 | 생각 | 감정 | Pain Point | 개선 기회 |
|---|---|---|---|---|---|
| **인지** | 네이버 카페 "스카이워커"에서 RAION 링크 클릭 | "또 광고성 학원 사이트 아냐?" | 의심 | 항공 정보 사이트 상당수가 학원 영업용. 신뢰도 낮음 | 랜딩 상단에 "광고 아님, 편집권 독립" 명시 + 출처 로고 노출 |
| **탐색** | 랜딩의 "오늘의 채용 TOP 3" 스크롤 | "내가 쓸 만한 공고가 있긴 하네" | 기대 | 승무원 공고인지 지상직인지 바로 구분 안 되면 이탈 | 공고 카드에 직군 태그(`승무원`·`지상직`) 큰 배지로 상단 고정 |
| **첫 사용** | 공고 상세 클릭 → 원본 사이트로 이동 | "다시 여기 돌아올 이유가 있나?" | 망설임 | 외부 링크 이동 후 RAION 재방문 동기 부재 | 공고 상세에 "비슷한 공고 3건" + "마감 임박 알림 받기" CTA |
| **전환** | 이메일 구독 폼 작성 (관심사 `승무원` 선택) | "이메일 인증까지 하는 게 귀찮긴 한데…" | 번거로움 | 더블 옵트인 이탈률 ↑. 인증 메일 못 찾음 | 구독 버튼 클릭 즉시 모달로 "1분 내 메일 확인하세요" 안내 + 스팸함 안내 |
| **유지** | 매일 아침 출근길에 다이제스트 수신 | "오늘은 뭐 있지?" | 기대·안정 | 공고 마감 지난 뉴스 섞이면 신뢰 훼손 | 다이제스트에 마감일·상태 배지 자동 표시 |
| **확산** | 친한 취준 친구 1명에게 URL 공유 | "이거 괜찮더라" | 만족 | 공유 동기 부족 (유니크 가치 체감 약함) | "이 공고는 카톡 발신 가능" 버튼 + 추천 시 알림 우선권 |

### 2.2 E2. 이준서 (Extreme) — 17세 항공 덕후·스포터

| 단계 | 행동 | 생각 | 감정 | Pain Point | 개선 기회 |
|---|---|---|---|---|---|
| **인지** | 디시 항공갤에서 누군가 "한국판 Simple Flying" 소개 | "Flightradar24 있는데 굳이?" | 회의 | 이미 영문 소스를 직독. RAION 차별점 불명 | 해외 뉴스 **한글 심층 요약 + 국내 맥락** 덧붙인 기사 예시 노출 |
| **탐색** | 운항 대시보드에 ICN 실시간 출발/도착 표 확인 | "필터는 없나?" | 실망 | 기종별·항공사별 필터 미제공. 표면적 정보만 | 고급 필터(기종·항공사·노선) 옵션 제공. "고급 모드" 토글 |
| **첫 사용** | 뉴스 카드에 `B777-300ER` 용어 → 툴팁 확인 | "이 정도는 나도 아는데, 더 깊게 쓸 수 있나?" | 지루 | 초보자용 용어 툴팁은 전문가에겐 불필요 | 로그인 시 "전문가 모드"로 툴팁 자동 숨김 |
| **전환** | 이메일 구독 (관심사 `항공기 기종`·`사건사고`·`정책`) | "광고성이면 바로 구독 해지할 거야" | 경계 | 스팸성 다이제스트 시 즉시 이탈 | 관심사 매칭 알고리즘 공개(투명성) + 구독자 피드백 버튼 |
| **유지** | 특이 기종 ICN 도착 시 푸시 받고 싶어함 | "이메일로는 너무 늦어" | 욕구불만 | 실시간성 부족 (이메일은 하루 1회) | **Phase 5 이후: 웹 푸시 알림** 또는 **Telegram 봇 연동** |
| **확산** | 인스타 계정(2천 팔로워)에 스크린샷 공유 | "여기 데이터 소스 신뢰할 만하네" | 자부심 | 공유 시 출처·크레딧 표기 없으면 재사용 위축 | OG 이미지에 "source: RAION Aviation Hub" 워터마크 자동 |

## 3. 두 여정의 패턴 차이 — UX 설계 시사점

| 차이점 | 김지원 (Core) | 이준서 (Extreme) | 결론 |
|---|---|---|---|
| **신뢰 진입 장벽** | "학원 광고 의심" | "영문 원문 대비 차별성 의심" | 둘 다 **"왜 RAION인가"**를 랜딩 첫 3초에 해결해야 함 |
| **정보 밀도 요구** | 낮음·명확함 원함 | 높음·심층 원함 | **토글 가능한 "전문가 모드"** 필요 |
| **전환 저항** | 귀찮음 | 경계심 | 구독 폼은 **"왜 구독해야 하는지" 3 bullet**로 해소 |
| **이탈 시점** | 공고 마감 후 재방문 이유 없음 | 실시간성 부족 | 각각 **"개인화 알림"** 으로 커버 (E2는 푸시, C1은 마감 임박 메일) |
| **확산 동기** | 친구 추천 | SNS 과시 | OG 이미지 + 직접 공유 버튼 필수 |

## 5. Adjacent / Non-user CJM은 왜 안 그리는가

- **Adjacent (A1~A3)**: Core 여정의 변형에 불과. 필요 시 C1 여정에 "교사 변형: 학생에게 보여줄 자료 다운로드 단계 추가"처럼 차이점만 메모.
- **Non-user (N1~N2)**: 여정을 그릴 필요 없음. 이들은 CJM 대상 자체가 아님. 리소스 분산 방지.
- **Extreme (E1 강철수)**: E2 이준서 하나로 "극한 정보 요구" 케이스는 충분. 필요 시 "시니어 UX" 이슈만 별도 메모 (OQ-R12).
