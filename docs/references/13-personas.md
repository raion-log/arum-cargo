# Personas — 아름 카고 (Arum Cargo) 페르소나 핀포인트

> **목적**: MVP 기능·UI·콘텐츠 결정 시 "이 사람이 이 화면을 보면 어떻게 행동할까"를 판단하기 위한 사고 도구.
> **전략**: **Pinpoint**. 단일 Core(C1)에 모든 결정의 기준점을 두되, 주변부(C2/C3/A1/B1)를 배제하지 않는 수준으로 유지.
> **구조**: Core 1 + Secondary 2 + Adjacent 1 + B-side 1 + Non-user 2 = **총 7 페르소나**.
> **주의**: 페르소나는 "평균"이 아니라 "대표"다. 각 페르소나는 실제 세그먼트의 의사결정 패턴을 단순화한 인물.

> **버전**: v0.3 (Cargo-First Pivot) · **근거**: [ADR-008](../adr/ADR-008-pivot-to-cargo-first.md)
> 이전 v0.2 페르소나(C1 김지원 승무원 외 11명)는 ADR-008로 인해 전면 교체되었다.

---

## 사용 방법

- 새 기능·화면을 설계할 때 **C1 기준 필수 통과** + C2/C3/A1/B1 중 최소 1명이 추가로 "쓸 만하다"고 판단되어야 통과
- PRD `User Stories` 섹션은 이 페르소나의 이름을 인용한다. 예: `"C1. 지훈은 아침 출근길에 Loadstar 요약을 5분 안에 훑고 싶다."`
- **Non-user는 건드리지 않을 사람 목록**. 리소스 낭비 방지용
- 핀포인트 전략상 **C1을 못 만족시키는 기능은 즉시 드롭**한다. "다른 사람이 좋아할 수도 있지 않을까"는 기각 사유가 됨

---

## 1️⃣ Core Persona — Pinpoint (핵심 1명)

MVP의 모든 결정이 여기서 시작된다. 구독자 500명의 **최소 60%** 이상이 C1 혹은 C1에 준하는 프로파일이어야 한다.

---

### C1. **이지훈** — 3년차 콘솔사 항공화물 영업·오퍼

| 항목 | 내용 |
|---|---|
| **직무/상태** | 콘솔사(Consolidator/Master Loader) 항공화물 영업·오퍼 3년차 (2~5년차 구간 중간값) |
| **나이** | 28세, 남성 |
| **위치** | 인천·김포 공항 상주 또는 마포·강남 본사 근무 |
| **기술 수준** | 중상. 회사 ERP·GMS(Global Messaging System)·카톡·네이버 능숙. 영어 메일 가능하지만 원서는 부담. |
| **연봉** | 4,200만원 내외 (인센 포함). 동종 업계 이직 시 +10~20% 기대 |
| **현재 Pain** | (1) 업계 운임·스페이스·공항 이슈 정보가 사내 카톡방에만 떠다님. (2) 이직 시 갈 만한 회사 정보 부족 — 사람인에 올라오는 공고는 대부분 "지인 추천만" 암묵. (3) 해외 매체(Loadstar, Air Cargo News) 영문 원서 읽기 부담. (4) 본인 커리어가 "계속 이 업계에서 맞나" 고민되는데 참고할 선배 사례·커뮤니티 부재. |
| **목표** | (1) 1~2년 내 더 큰 콘솔사 또는 대형 포워더(판토스·CJ대한통운·한진)로 이직. (2) 업계 운임·이슈를 아침 출근길에 5분 안에 파악. (3) "나만 모르는 거 아닌가" 불안 해소. |
| **사용 상황** | 출근길 7:00~7:30 지하철/버스에서 모바일로 이메일 체크. 퇴근 후 21:00~22:00 집 노트북으로 채용 공고 서핑. |
| **대체 솔루션 (현재)** | 카고프레스(한정적), 사람인 알림, 사내 카톡방, 네이버 뉴스 `항공화물` 검색, 지인 전화 |
| **기대 개선점** | "매일 아침 한 번에 업계 뉴스 요약 + 오늘의 주요 채용 2~3건 + 에디터 Pick으로 현직자 시선" |
| **아름 카고 가치 연결** | I-Side 일일 다이제스트(`화물·운임·공항·정책` 태그) + A-Side 콘솔/포워더 경력직 공고 + 에디터 Pick + Phase 5.5 `/employers` 정보 |

**왜 이 사람이 Core인가**:
- 2~5년차는 "배운 건 많고 바꾸고 싶은 것도 많은 구간" → 외부 정보에 대한 지불 의사(시간 지불) 가장 높음
- 이직 전 정보 탐색이 활발 → 채용 섹션 재방문률 최고
- 본인 업무 중 정보 비대칭에서 오는 Pain을 구체적으로 말할 수 있는 유일한 구간 (신입은 모르고, 시니어는 이미 네트워크로 해결)
- 창립자(11년차 현직자)와 **정서적 거리가 가장 짧음** → 에디터 Pick 톤이 자연스럽게 통함

## 페르소나 활용 체크리스트 (PRD 작성 시)

- [ ] 새 기능 → **C1이 쓴다**로 판단되는가? (NO면 드롭 또는 Phase 5.5+로 유예)
- [ ] 새 기능 → C2/C3/A1/B1 중 최소 1명이 추가로 이득을 보는가
- [ ] 새 화면 → C1이 모바일 5분 안에 핵심 가치를 받는가
- [ ] 새 카피 → C2 박서연(1년차 신입)이 한 번 읽고 이해하는가
- [ ] 이메일 다이제스트 → C1이 출근길 지하철에서 5분 안에 훑는가
- [ ] 에디터 Pick → C1이 "우리 업계 사람이 쓴 글" 이라고 느끼는가 (C2는 "배움이 있다", C3은 "공유할 만하다"고 느끼는가)
- [ ] 번역 뉴스 → C1이 원문 대신 이 요약으로 충분하다고 느끼는가
- [ ] `/employers` 기능 (Phase 5.5) → B1이 문의 남길 이유가 있는가
- [ ] **Non-user는 무시**: N1·N2를 위한 기능은 만들지 않는다
