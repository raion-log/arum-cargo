## 1. External Environment & Niche (산업 분석 및 플랫폼 기회)

### 1.1 거시 환경
- **디지털 혁신 지체**: 항공사·기존 업계가 레거시 시스템에 지나치게 의존. '디지털 프로덕트 매니저' 부재
- **가시성·자동화 수요 급증**: 전자상거래 급성장 → 화물 추적·가시성·프로세스 자동화 요구 폭발
- **RPA, RTTVPs, API 기술**이 2025년 이후 물류 경쟁력의 핵심으로 지목됨
- **AI 투자 확대**: 데이터센터·피지컬 AI에 대한 천문학적 투자 지속

### 1.2 틈새 시장
항공업계가 자체적으로 해결하지 못하는 다음 두 가지를 RAG 기반 AI + API 연동으로 즉각 해결:
1. **채용·교육의 파편화**
2. **B2B 물류 정보의 비대칭성**

**서드파티 통합 플랫폼**의 수요가 크다.

## 2. Value Proposition (서비스 구조화)

### 2.1 A-Side — Academy & Career
- 승무원·지상직·조종사 등 **직군별 합격 후기 및 채용 DB**
- 취준생 조건에 맞는 **이미지 메이킹 스마트 매칭** (메이크업 / 사진 스튜디오)

### 2.2 L-Side — Logistics & Cargo
- 항공사·포워더·콘솔사 간 **실시간 단가·컨택 연계 허브**
- AI 활용 **항공 화물 필수 서류 자동 검토·첨삭** (AWB 등)

### 2.3 I-Side — Information & Community
- 전 세계 공항 **실시간 운항·슬롯 API 연동**
- AI 비서가 매일의 **글로벌 항공·물류 정책 변화 뉴스 요약** 대시보드

## 3. BM Design (비즈니스 모델 설계)

RAION의 수익은 **3축 + 3모델**로 구성된다.

### 3.1 제휴·중개 수수료 (B2C Affiliate) — A-Side
- 스마트 매칭을 통한 **제휴 학원·스튜디오·메이크업 샵 예약 수수료**
- 예약당 수수료율: 업체별 협상 (벤치마크 5~15%)

### 3.2 프리미엄 채용 솔루션 (B2B HR) — A-Side 확장
- 지상직·화물 전문가 **특화 직군 헤드헌팅 매칭**
- **채용 성공 보수** (연봉의 일정 %) + **프리미엄 공고료**

### 3.3 SaaS + 데이터 구독 (B2B Data) — L-Side / I-Side
- 포워더·화주용 **운임 추이 시각화 대시보드**
- **AWB 서류 자동화 툴**
- **실시간 항공 데이터 API**
- 유료 멤버십 기반 구독 모델

### 3.4 수익화 타임라인 (현재 기획 기준)

| 시기 | 도입 수익원 |
|---|---|
| 2026 H1 (MVP) | ❌ 수익화 없음. 트래픽·이메일 구독자 확보 집중 |
| 2026 H2 | ✅ 제휴 수수료 (A-Side Affiliate) 시작 |
| 2027 H1 | ✅ B2B HR 프리미엄 공고료 |
| 2027 H2 | ✅ L-Side SaaS 구독 |

> ⚠️ 이 타임라인은 **가설**이다. 실제 수익화 시점은 MVP 런칭 후 이메일 구독자 수 + 제휴 업체 확보율에 따라 조정된다.

## 4. Risk & Mitigation (리스크 및 대응)

상세: [08-macro-risks-2026.md](./08-macro-risks-2026.md)

### 4.1 거시경제 리스크
- 2026년 한국 경제: **2.0% 저성장**, **1,400원대 중반 고환율**, **미국 관세 조치**

### 4.2 대응 기제
- **반도체·IT 수출은 2026년에도 견조한 성장** → 단가 방어 용이한 **고부가가치(IT·반도체) 항공 화물 네트워크** 집중
- **B2C(채용/교육) + B2B(물류/데이터)** 포트폴리오 다각화로 경기 침체 시 현금 흐름 변동성 방어

## 5. Execution Principles (실행 원칙)

### 5.1 완결형 여정 설계
> 플랫폼은 단순 게시판/정보 나열을 넘어, **RAG 기반 AI 에이전트**를 통해 사용자의 맥락을 인지하고 즉각적인 거래(예약·서류 검토·채용 지원)로 이어지는 **완결형 여정(User Journey)** 을 설계해야 한다.

### 5.2 기술적 해자 (Moat) 구축
> 항공 화물·채용 도메인 지식을 갖춘 **10년 차 이상의 내부 전문성**을 AI 아키텍처(서류 자동 검토, 맞춤형 챗봇)에 학습시켜, 범용 IT 기업이 모방할 수 없는 **강력한 기술적 해자**를 구축해야 한다.

### 5.3 단계적 애자일 개발
> **2026년 상반기 내** 채용과 이미지 메이킹 업체를 연계하는 **B2C 최소 기능 제품(MVP)** 을 최우선으로 런칭하여 초기 트래픽을 확보하고, 이후 **API 기반 B2B 물류 대시보드**로 확장하는 **단계적(Phased) 애자일 개발 방법론** 을 엄격히 준수해야 한다.
