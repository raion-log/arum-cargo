# Problem Statements — 3개 영역 문제 정의

> 출처: 대화 기반 산업·시장 분석 3편(A/L/I) + Genspark PDF p.5 Problem Statement.
> 각 Side가 해결하려는 **실제 유저의 불편함**을 "페르소나 / 상황 / 고통" 3축으로 명시.

---

## 1. A-Side — B2C 구직자 (승무원·지상직 취준생)

### 1.1 시장 분석
- **거시 트렌드**: 한류 확장 및 K-소비재 수출 증가에 따른 항공 수요 회복세에도 불구하고, **교육·채용 시스템은 여전히 레거시 의존**
- **경쟁**: 오프라인 학원 중심의 파편화, 브랜드 충성도 낮음, **버티컬 플랫폼 부재**
- **구매자 교섭력**: 약함 — 원스톱 여정 통합 플랫폼에 의존도 빠르게 상승할 구조
- **핵심 가치 창출**: 맞춤형 채용 공고 큐레이션 / 메이크업·스튜디오 제휴 스마트 매칭 / 직군별 합격 후기 DB

### 1.2 문제 정의 (Jobs-to-be-Done 형식)

> **[항공업계(승무원/지상직 등) 취업 준비생]** 이
> **[면접 복장, 메이크업, 프로필 촬영 등을 개별적으로 예약하고 채용 정보를 탐색하는 상황]** 에서 겪는
> **[파편화된 오프라인 정보로 인한 탐색 비용 증가와 커리어 준비 과정의 단절이라는 불편함]** 을 해결하는 것이 중요한 문제이다.

### 1.3 고통 지표 (Genspark PDF p.5 인용)
- **68%** — 정보 탐색 시간
- **42%** — 비효율적 과정
- **3.2x** — 비용 증가

### 1.4 핵심 해결 방향
- **원스톱 여정** 제공 (채용 정보 → 이미지 메이킹 → 면접 준비가 한 플랫폼에)
- **직군별 합격 후기** 로 정보 비대칭 해소
- **이메일 구독** 으로 능동적 정보 전달

---

## 2. L-Side — B2B 포워더·화주 (항공 화물 시장, **MVP 스코프 밖**)

### 2.1 시장 분석
- **거시 트렌드**: 전자상거래 급성장 + 미 관세 인상 등 공급망 재편 → **물류 가시성 및 DX 요구 폭발적 증가**
- **경쟁**: 낮음 — 여전히 이메일·전화 기반 수작업 관행
- **공급자(항공사) 교섭력**: 매우 높음 — 데이터 통제력 압도적
- **대체재 위협**: 낮음
- **핵심 가치 창출**: 항공사-포워더 실시간 단가·컨택 연계 / RAG 기반 AWB 서류 자동 검토 첨삭

### 2.2 문제 정의

> **[고부가가치(IT·반도체) 항공 화물을 취급하는 중소 포워더 및 화주]** 가
> **[다양한 항공사의 운임 단가를 비교하고 복잡한 수출입 필수 서류(AWB 등)를 작성해야 하는 상황]** 에서 겪는
> **[항공사별로 파편화된 정보 비대칭성과 수작업 서류 처리로 인한 리드타임 지연 및 치명적 휴먼 에러 발생이라는 불편함]** 을 해결하는 것이 중요한 문제이다.

### 2.3 고통 지표 (Genspark PDF p.5 인용)
- **45%** — 수작업 비율
- **2.5x** — 처리 시간
- **28%** — 오류율

### 2.4 핵심 해결 방향
- **RAG 기반 AWB 자동 검토** — 10년차 현업 지식 학습한 AI가 오류 첨삭
- **실시간 단가 대시보드** — 항공사·포워더 간 정보 비대칭 해소
- **RPA 연계** — 세관 신고·규정 준수 자동화

### 2.5 MVP 스코프 밖 유지 이유
- 도메인 복잡도 높음
- 항공사 파트너십이 선행되어야 함
- **MVP는 A+I Side 트래픽 확보 후 레버리지로 활용**

---

## 3. I-Side — 정보·커뮤니티 (항공 업계 종사자·기획자)

### 3.1 시장 분석
- **거시 트렌드**: 글로벌 AI 인프라 투자 지속 + 항공업계 인력 부족·비효율 → **지능형 의사결정 솔루션 도입 가속**
- **경쟁**: 중간 — 빅테크 진입 리스크 상존하나 도메인 지식 부재가 높은 진입 장벽
- **공급자(항공사·언론) 통제력**: 중간
- **핵심 가치 창출**: 전 세계 공항 운항/슬롯 API Seamless 연동 / 10년차 도메인 지식 RAG AI 자산화 / 매일의 글로벌 항공 정책·뉴스 자동 요약 대시보드

### 3.2 문제 정의

> **[현업 항공업계 종사자 및 관련 비즈니스 기획자]** 가
> **[글로벌 항공·물류 정책 변화와 실시간 운항/슬롯 데이터를 분석하여 신속한 의사결정을 내려야 하는 상황]** 에서 겪는
> **[레거시 시스템의 폐쇄성으로 인한 데이터 수집의 한계와, 전문 지식이 결여된 범용 IT 도구 사용으로 인한 부정확하고 파편화된 정보 취득의 불편함]** 을 해결하는 것이 중요한 문제이다.

### 3.3 핵심 해결 방향
- **단일 대시보드** — 뉴스 + 운항 + 정책을 한 화면에
- **AI 요약** — 긴 외신 기사를 한국어 핵심 요약으로 제공
- **출처 투명성** — 모든 정보에 원본 링크 노출 → 신뢰 확보

---

## 4. 3-Side 공통 가치 제안

세 영역 모두 **"파편화된 정보를 AI로 통합하여 단편적 조회를 넘어 완결된 여정으로 전환한다"** 는 공통 철학을 공유한다.

| Side | 파편화 대상 | 통합 방식 | 완결 지점 |
|---|---|---|---|
| A | 채용공고 + 이미지 메이킹 업체 정보 | 직군별 큐레이션 + 스마트 매칭 | 예약·지원 완료 |
| L | 항공사 단가·AWB 서류 | 실시간 단가 매칭 + 서류 자동 검토 | 계약·선적 완료 |
| I | 운항·뉴스·정책 | API Hub + AI 요약 | 의사결정 액션 |

---

## 5. 해결 방안 요약 (Genspark PDF p.5 인용)

> **해결 방안**: RAG 기반 AI 에이전트 + 통합 API 대시보드로 심리스하게 연계 → **완결형 서비스 제공**
