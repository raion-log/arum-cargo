## 2. A-Side (Academy & Career)

### 2.1 타겟
- 승무원 취준생 (국내항공사, 외항사)
- 지상직 취준생
- 조종사 준비생
- 항공 화물 전문가 지망생 (Phase 2 연계)
- 경력 전환자

### 2.2 핵심 기능 (MVP)

| 기능 | 설명 |
|---|---|
| **직군별 채용 DB** | 승무원 / 지상직 / 조종사 / 기타 직군 태그 필터. 항공사·LCC·외항사 그룹핑 |
| **합격 후기 섹션** | 직군·회사별 후기 수집. Phase 1 MVP에서는 큐레이션 에디터용 |
| **스마트 매칭 (제휴 중개)** | 면접 복장 / 메이크업 / 프로필 스튜디오 / 면접 코칭 업체 추천. 예약 시 수수료 수취 |
| **AI 챗봇 (Phase 4)** | RAG 기반. "승무원 메이크업 예약하고 싶어" → 위치 기반 제휴 업체 검색 → 예약 링크 |

### 2.3 수익 모델
- **제휴 중개 수수료 (Affiliate)**: 제휴 업체 예약당 수수료
- **프리미엄 채용 솔루션 (B2B HR)**: 특화 직군 헤드헌팅 성공 보수 + 프리미엄 공고료
- **AI 매칭 정확도 목표**: 85%

### 2.4 MVP 범위
- ✅ 채용 DB + 직군 필터 + 카드 UI
- ✅ 제휴 업체 목록 (초기 3~5곳 수동 큐레이션)
- ❌ AI 챗봇 (Phase 4로 유예)
- ❌ 실제 예약·결제 (Phase 2)

## 3. L-Side (Logistics & Cargo) — Phase 2, MVP 스코프 밖

### 3.1 타겟
- 중소 포워더
- 항공 화물 담당자
- IT·반도체 수출 화주
- 항공사 화물 사업부

### 3.2 핵심 기능 (Phase 2 계획)

| 기능 | 설명 |
|---|---|
| **B2B 실시간 단가 매칭** | 항공사·포워더·콘솔사 간 단가 동향 + 컨택 포인트 |
| **AWB 서류 자동 검토** | RAG AI + Computer Vision으로 AWB 오류 자동 첨삭 |
| **RPA 연계** | 세관 신고, 규정 준수 검사 자동화 |
| **시황 대시보드** | 고부가가치 화물 운임 추이, 용량 현황 |

### 3.3 수익 모델
- **B2B SaaS 구독료** (월·연 멤버십)
- **기업용 API 접근료**
- **고부가가치 계약 중개 수수료**

### 3.4 MVP에서 제외된 이유
- 도메인 복잡도 ↑, 항공사 파트너십 선행 필요
- 초기 B2C 트래픽 확보 후 협상 레버리지로 활용 예정
- **단, PRD와 데이터 모델 설계 시 Phase 2 확장성은 염두에 둔다**

## 4. I-Side (Information & Community)

### 4.1 타겟
- 항공업계 현업 종사자
- 항공·물류 비즈니스 기획자
- 일반 항공 관심 유저 (확장)
- A-Side 취준생 (크로스 플로우)

### 4.2 핵심 기능 (MVP)

| 기능 | 설명 |
|---|---|
| **항공 뉴스 카드** | 네이버 뉴스 + 국내 화물 매체 RSS + 외국 뉴스 번역. 출처 표기 필수. 카드 UI |
| **실시간 운항 대시보드** | 주요 공항(ICN 우선) 출발/도착 + 지연 현황. API 캐싱 |
| **AI 정책 요약** | 글로벌 항공·물류 정책 변화 일일 요약 (Phase 4 LLM 연동) |
| **슬롯(Slot) 정보** | 주요 공항 슬롯 API 연동 (Phase 4) |

### 4.3 수익 모델
- **프리미엄 데이터 구독** (Phase 2 이후)
- **B2B API 접근료**
- **MVP 단계에서는 광고·수익화 없음. 트래픽·이메일 구독 수집이 목표**

### 4.4 MVP 범위
- ✅ 뉴스 카드 (네이버 API + RSS)
- ✅ 간이 운항 대시보드 (AviationStack / OpenSky 무료 티어)
- ✅ AI 정책 요약 (Phase 4, OpenAI 연동 시)
- ❌ 실시간 슬롯 상세 (항공사 API 계약 필요)

## 5. 공통 기반 레이어

### 5.1 RAG AI Engine
- 10년차 이상 현업 전문성 학습
- AWB 서류 규칙, 규제 가이드, 항공 용어 등
- **MVP에서는 기본 LLM(OpenAI) 사용. 커스텀 RAG는 Phase 2에서 구축**

### 5.2 통합 API Hub
- 외부 API 어댑터 레이어 (`src/lib/api/*`)
- 캐싱·레이트 리밋 관리
- 에러 폴백 → Mock 데이터 or 캐시

### 5.3 데이터 파이프라인
- Supabase PostgreSQL
- 뉴스 스크래핑 cron (Vercel Cron)
- 채용 공고 동기화 cron

## 6. Cross-Side Synergy

A / L / I 는 독립이 아니라 **서로의 유저와 데이터를 공유**해야 한다:

| From → To | 시너지 |
|---|---|
| **I-Side → A-Side** | "오늘의 항공 뉴스"를 보러 온 유저 → 채용 CTA 노출 |
| **A-Side → I-Side** | 취준생이 이메일 구독하면 관심 항공사 뉴스 선제 제공 |
| **A-Side → L-Side (P2)** | A-Side의 "항공 화물 전문가 지망" 세그먼트 → L-Side 파트너 후보 DB |
| **L-Side → I-Side (P2)** | L-Side 단가·운임 데이터 → I-Side 프리미엄 시황 대시보드 |

> **이메일 구독자의 관심사 태그(subscribers.interests[])가 이 시너지의 핵심 스위치.**
