# Value Chain Transformation — As-Is vs To-Be

> 출처: NotebookLM Blueprint p.4, Genspark PDF p.3.
> 기존 파편화된 가치사슬과 RAION AI 통합 이후 가치사슬의 대비.

## 1. As-Is — 파편화된 기존 가치 사슬

```
┌──────────────────┐   ❌    ┌──────────────────┐   ❌    ┌──────────────────┐
│   투입/조달      │ ──────→ │   운영/마케팅    │ ──────→ │    서비스        │
│                  │ [끊김]  │                  │ [끊김]  │                  │
│ 수작업 기반      │         │ 높은 인적 의존도 │         │ 정보 제공에      │
│ 스페이스 확보    │         │ 비효율적 매칭    │         │ 그치는           │
│ 및 오프라인      │         │                  │         │ 단편적 여정      │
│ 정보 수집        │         │                  │         │                  │
└──────────────────┘         └──────────────────┘         └──────────────────┘
  [FRAGMENTED]                  [HIGH_LATENCY]              [FRAGMENTED]
```

### 문제점
- **3단계 모두 끊어져 있음** — 데이터·프로세스·유저 여정이 모두 분절
- 수작업 비중 높음 → 에러·리드타임 증가
- 최종 유저는 **단편적 정보**만 받고, 실제 거래(예약·계약)로 이어지지 못함

## 2. To-Be — RAION AI 통합 가치 사슬

```
┌──────────────────┐   ✅    ┌──────────────────┐   ✅    ┌──────────────────┐
│    Inbound       │ ──────→ │   Operations     │ ──────→ │    Service       │
│                  │         │                  │         │                  │
│ 정부 및 항공사   │         │ 거시 리스크      │         │ 단순 조회 넘어   │
│ 레거시 시스템과  │         │ 방어용 고부가    │         │ 예약/실행까지    │
│ Seamless         │         │ 가치(IT·반도체)  │         │ 완결 짓는        │
│ API 연동         │         │ 화물 네트워크    │         │ AI 에이전트      │
│                  │         │ 최적화           │         │ End-to-End 여정  │
└──────────────────┘         └──────────────────┘         └──────────────────┘
  [API_INTEGRATED]              [OPTIMIZED]                 [AI_DRIVEN]
        ↑
  ┌───────────────┐
  │ 관계 부처 합동 │ (정부 협업 채널)
  └───────────────┘
```

### 핵심 변화

| 단계 | As-Is | To-Be |
|---|---|---|
| Inbound | 수작업 스페이스 확보 | API로 항공사·정부 시스템 자동 연동 |
| Operations | 인력 매칭 | 거시 리스크 방어형 고부가가치 화물 집중 |
| Service | 정보 제공에 그침 | AI 에이전트가 예약·실거래까지 완결 |

## 3. 산업 핵심 기업별 Value Chain 포지션

> 출처: Genspark p.3 — "산업 핵심 기업 Value Chain 분석"

### 3.1 공급자: 대한항공 (Major Carrier, 강함)
- ✅ 글로벌 노선 인프라 운영 역량 압도적
- ⚠️ 독점적 데이터 폐쇄성 → API 연동 갭 발생

### 3.2 수요자: CJ대한통운 (Global Forwarder, 강함)
- ✅ 조달 물류·구매력 뛰어남
- ⚠️ 항공사별 파편화된 단가 정보 + AWB 처리 부재

### 3.3 경쟁자: ANC 승무원 학원 (A-Side Incumbent, 중간)
- ✅ 대면 컨설팅 + 항공사 네트워크 강점
- ⚠️ 데이터 기반 매칭 로직 등 디지털 인프라 취약

### 3.4 잠재 진입자: 네이버 / 카카오 (Big Tech, 중간)
- ✅ 최고 수준의 초거대 AI 인프라 보유
- ⚠️ 항공·물류 현장의 10년차 도메인 전문성 결여

### 전략적 시사점
> 빅테크가 침투하지 못하는 **도메인 지식을 AI화**하고,
> 공급자와 수요자 간의 **데이터 단절을 잇는 브릿지(Bridge)** 역할을 수행해야 함.

## 4. RAION의 가치사슬 개입 지점

RAION은 **가치사슬 전 단계에 걸쳐 개입**하지만, 각 단계별로 진입 우선순위가 다르다:

| 단계 | Phase 1 (MVP) | Phase 2 | Phase 3 |
|---|---|---|---|
| Inbound | 공공 API 활용 (워크넷, 네이버 뉴스) | 항공사 파트너십 API 연동 | 정부 관계부처 협업 |
| Operations | A-Side 제휴 매칭 (수동 큐레이션) | L-Side 고부가 화물 매칭 (AI) | 통합 AI 에이전트 |
| Service | 공개 대시보드 + 이메일 구독 | B2B SaaS 구독 | 자연어 명령 인터페이스 |

### MVP의 가치 사슬 참여
```
[네이버 뉴스 / 워크넷 / AviationStack]
            ↓ API 수집
[Next.js 서버 (API Routes)]
            ↓ 캐싱·정제
[Supabase DB]
            ↓ 렌더링
[공개 대시보드] ──→ [이메일 구독] ──→ [Resend 일일 다이제스트]
                                           ↓
                                    [재방문 / 시너지 락인]
```
