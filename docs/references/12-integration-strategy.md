# Integration + Value-Add Strategy — 통합 + 부가가치 전략

> **핵심 질문**: "이미 사람인·네이버뉴스·AviationStack 같은 서비스가 존재하는데, RAION이 단순히 그걸 모아 보여주는 게 의미가 있는가?"
> **답**: **"Aggregator + AI Value-Add + Owned Audience"** 3층 구조로 차별화한다. 단순 통합(Aggregator)만으로는 Moat가 없다. 가치를 얹고, 독자 DB를 소유해야 한다.

---

## 1. 문제 인식

### 현재 항공 정보 파편화
- **채용**: 사람인, 잡코리아, 원티드, 워크넷, 각 항공사 채용 페이지 → **8곳 이상 분산**
- **뉴스**: 네이버뉴스, 각 언론사, 해외 Simple Flying / Aviation Week → **언어·매체별 분리**
- **운항 데이터**: 인천공항 홈페이지, 각 항공사 앱, Flightradar24, AviationStack → **UX 제각각**
- **커뮤니티**: 네이버 카페 (스카이워커 등), 디시 항공갤, 학원 블로그 → **신뢰도 편차**
- **용어·지식**: 블로그·카페에 산발, 학원 내부 자료 → **체계 없음**

**사용자 페인**: 항공 취준생·업계 종사자는 **하루 1~2시간을 "정보 찾기"에만 낭비**한다. (05-problem-statements.md의 68% 수치 참조)

### 단순 통합(Pure Aggregator)의 함정
- 법적 리스크 (뉴스 재배포, API ToS)
- 원본 사이트 대비 **차별화 부족** → 유저는 결국 원본으로 돌아감
- 광고 외 수익 모델 부재
- **대표 실패 사례**: 네이버 뉴스 재생산 블로그들, 구글뉴스 카피캣들

## 2. RAION 3층 전략

```
┌─────────────────────────────────────────────────┐
│ Layer 3: OWNED AUDIENCE (독자 소유층)            │
│ ───────────────────────────────────────────     │
│ 이메일 구독자 DB + 관심사 세그먼트 + 재방문 루프  │
│ → 유일한 Moat. 타사가 복제 불가능.               │
├─────────────────────────────────────────────────┤
│ Layer 2: AI VALUE-ADD (부가가치층)               │
│ ───────────────────────────────────────────     │
│ • 항공 특화 큐레이션 (키워드·도메인 필터)         │
│ • 외국 뉴스 한글 요약 (LLM 번역)                │
│ • 용어 툴팁 (AviationTerm 컴포넌트)             │
│ • 관심사 기반 개인화 다이제스트                   │
│ • 통합 대시보드 UX (한 화면에 뉴스+운항+채용)    │
├─────────────────────────────────────────────────┤
│ Layer 1: AGGREGATOR (통합층)                    │
│ ───────────────────────────────────────────     │
│ 네이버뉴스 API / AviationStack / 워크넷 /       │
│ Simple Flying RSS / 국내 화물 매체 RSS          │
│ → API Symbiosis. 저비용·빠른 구축.              │
└─────────────────────────────────────────────────┘
```

---

## 3. 각 층의 역할과 Moat 기여도

### Layer 1 — Aggregator (통합층)
**역할**: 원본 데이터를 모으는 파이프라인. **"속도와 폭"** 경쟁.

**구성**:
- 뉴스: 네이버 뉴스 API + 국내 화물 매체 RSS + 해외 항공 뉴스 RSS
- 운항: AviationStack / OpenSky (무료 티어)
- 채용: 워크넷 OpenAPI + (Phase 2) 사람인 OpenAPI
- 정책·공공: 국토부 보도자료 RSS, 항공진흥협회

**Moat 기여도**: ⭐ (낮음)
- 누구나 같은 API를 호출할 수 있음.
- 단, **"얼마나 잘 조합했는가"**는 차별화 여지 있음 (예: 네이버뉴스 + RSS + 번역을 한 카드로 묶기).

**전략**: **"싸고 빠르게 최대한 많이"**. Phase 4에서 5개 이상 소스 동시 연동.

---

### Layer 2 — AI Value-Add (부가가치층)
**역할**: 원본 데이터에 **의미·맥락·편집**을 덧씌워 "RAION에서만 볼 수 있는 형태"로 변환.

**구성**:

#### 2.1 항공 도메인 필터링
- 원본 뉴스 수천 건 → 항공 키워드(`대한항공`, `LCC`, `항공화물`, `보잉`, `슬롯` 등)로 자동 필터
- 출처 신뢰도 점수 (주요 언론사 > 블로그)
- 중복 제거 (같은 사건의 여러 기사 → 대표 1건 + 관련 N건)

#### 2.2 다국어 요약 파이프라인
- Simple Flying / Aviation Week 영어 원문 → GPT-4o-mini로 2~3문장 한글 요약
- 원문 링크는 그대로 유지 (저작권 준수)
- 비용: 월 $0.1 이하 (09-news-sources.md 참조)

#### 2.3 용어 해설 (Aviation Glossary 연동)
- 뉴스 본문에 `AWB`, `Slot`, `RTTVPs` 같은 용어가 나오면 자동으로 `<AviationTerm />` 툴팁 주입
- 10-aviation-glossary.md의 30개+ 용어가 시드

#### 2.4 관심사 기반 개인화
- 구독 시 3~5개 관심사 태그 선택 (예: `승무원`, `화물`, `LCC`, `정책`)
- 일일 다이제스트는 **각 구독자마다 다른 기사 조합** 발송
- → 재방문 루프 + 구독 유지율 ↑

#### 2.5 통합 UX (단일 대시보드)
- 한 화면에 `오늘의 뉴스 TOP 5` + `실시간 ICN 운항 현황` + `신규 채용 TOP 3` 동시 노출
- 경쟁 서비스는 이게 다 분리되어 있음

**Moat 기여도**: ⭐⭐⭐ (중간)
- LLM 기술은 복제 가능. 하지만 **"항공 도메인에 튜닝된 큐레이션 룰"**은 도메인 지식 축적이 필요.
- 용어집 + 필터 룰은 시간이 갈수록 정교해짐.

**전략**: Phase 3~4에서 기본 형태 구축, Phase 5 이후 구독자 피드백으로 지속 개선.

---

### Layer 3 — Owned Audience (독자 소유층) ★ 진짜 Moat
**역할**: 이메일 구독자 DB. **타사가 절대 복제할 수 없는 자산**.

**왜 Moat인가**:
1. **직접 소통 채널**: 알고리즘 의존 없이 구독자 인박스에 직접 도달.
2. **세그먼트 데이터**: 누가 무슨 관심사인지 알고 있음 → B2B 레버리지 가능.
3. **스위칭 코스트**: 구독자는 한 번 구독하면 쉽게 이탈 안 함.
4. **네트워크 효과**: 구독자 수가 곧 B2B 협상력(제휴 스튜디오, 광고주, 학원).

**활용 경로**:
- **B2C**: 일일 다이제스트 → 재방문 → 유료 프리미엄 기능(실시간 알림, 전문 리포트)
- **B2B**: "RAION은 항공 관심층 2,000명을 보유" → 제휴 스튜디오·학원·채용 대행사에 광고·후원 제안
- **데이터**: 어떤 기사·채용이 클릭율 높은지 → 업계 인사이트 리포트로 재가공

**Moat 기여도**: ⭐⭐⭐⭐⭐ (최고)

**전략**: Phase 5에서 **첫 500명**을 최우선 목표로. 단순 트래픽(MAU)보다 **구독 전환율**을 KPI 1위로.

## 4. 경쟁 구도 분석 (누가 비슷한 걸 하는가)

| 경쟁군 | 예시 | 강점 | 약점 / RAION 차별점 |
|---|---|---|---|
| 종합 채용 플랫폼 | 사람인, 잡코리아 | 공고 수, 브랜드 | 항공 특화 X, 뉴스 없음, 용어·커리어 가이드 부재 |
| 포털 뉴스 | 네이버 뉴스 | 방대한 기사량 | 항공 큐레이션 X, 채용·운항 통합 없음 |
| 운항 추적 | Flightradar24 | 실시간성, 정확도 | 한국어 X, 뉴스·채용 없음, 유료 위주 |
| 항공 전문 뉴스 | Simple Flying, Aviation Week | 심층 콘텐츠 | 영어, 한국 시장 맥락 부족 |
| 업계 커뮤니티 | 네이버 카페 "스카이워커" | 유저 결집, 경험 공유 | 정보 신뢰도 편차, UX 구식, 구조화 부재 |
| 학원·아카데미 블로그 | 각 승무원 학원 | 취준 노하우 집약 | 영업용 편향, 뉴스·운항 부재 |
| 공공 API | 워크넷, 국토부 | 공신력, 무료 | UX 없음, 모바일 최적화 X |

**RAION의 유일성**: 
- 위 7개 카테고리의 **"항공 관련 부분만 선별적으로 통합 + 한국어 큐레이션 + 구독 기반 개인화"**를 동시에 제공하는 서비스는 **현재 부재**.
- 해외에는 Simple Flying(영문), Points Guy(여행 중심) 유사 사례 있음. 한국 시장에는 빈 자리.

## 5. 리스크 및 대응

| 리스크 | 가능성 | 대응 |
|---|---|---|
| API ToS 변경으로 소스 끊김 | 중 | 다중 소스 구조 (한 소스 죽어도 다른 소스로 폴백). `docs/prd/04-api-integration.md`에 폴백 전략 명시 예정 |
| 사람인·네이버가 똑같은 기능 추가 | 낮음 | 대기업은 항공 버티컬 진출 ROI 낮음. 단, L-Side(B2B)까지 확장 시 주의 |
| 뉴스 재배포 저작권 소송 | 중 | "제목+요약+원문링크+출처"만. 변호사 자문 (OQ-R4) |
| 번역 품질 이슈 | 낮음 | LLM 번역 후 수동 스팟 체크, 원문 링크 병기 |
| 구독자 이탈률 높음 | 중 | 개인화 + 관심사 기반 → 클릭률 추적 → 지속 개선 |
| Aggregator 브랜드 이미지 (카피캣 인식) | 중 | **에디토리얼 톤** + 자체 해설·인터뷰 콘텐츠 추가로 "언론사" 이미지 강화 |
